### HTML & Semantics
1. **What is semantic HTML and why is it important in your SWIGGY project?**

3. **What is the difference between block and inline elements?**
   - Expected Answer: Block elements take full width (div, section, p), while inline elements only take necessary width (span, a, img). You've used both in the SWIGGY layout for responsive design.

5. **Explain the box model in CSS.**
   - Expected Answer: Content → Padding → Border → Margin. Important for calculating element widths/heights and spacing in your restaurant cards and cart layout.
   - box model ----  it is the fundamental concept in web developement.
   - it contains  the content -- padding --- border--- margin
   - content  is the actual  coontent of the box(text,  image, etc)
   -  padding is  creates  space around the content(element)
   -  border is  applied to the surrounding  padding and content.
   -   margin  is clears the space outside the border.

15. **Explain event delegation and how you implemented it in SWIGGY.**
    - Expected Answer: Attach event listener to parent element instead of individual children. Efficient for handling clicks on dynamically added items:
      ```javascript
      cartContainer.addEventListener('click', (e) => {
        if (e.target.classList.contains('remove-btn')) {
          removeFromCart(e.target.dataset.itemId);
        }
      });
      ```
- Add one listener to a parent and use `event.target` to detect which child was interacted with. Works because events bubble.

Why use it?
- Better performance (fewer listeners).
- Works with dynamic children added later.

16. **What is the difference between synchronous and asynchronous JavaScript?**
    - Expected Answer:
      - Synchronous: Blocks execution until complete (loops, calculations)

28. **What is debouncing and throttling? Where would you apply them in SWIGGY?**
    - Expected Answer:
      - Debouncing: Wait until user stops typing (search)
      - Throttling: Limit function calls per time interval (scroll events)
      ```javascript
      function debounce(func, delay) {
        let timeoutId;
        return function(...args) {
          clearTimeout(timeoutId);
          timeoutId = setTimeout(() => func(...args), delay);
        }
      }
      ```


## 📋 Executive Summary

Developed and deployed a fully functional food delivery platform frontend that replicates the user experience of Swiggy. The application features dynamic restaurant discovery, shopping cart management with Redux state management, and responsive design across all devices. Successfully deployed to production on Vercel with continuous integration/deployment pipeline.
